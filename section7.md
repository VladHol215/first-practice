## Розділ 7. Модель даних системи

### 7.1. Логічна структура бази даних
Для забезпечення цілісності та швидкодії системи онлайн-банкінгу обрано реляційну модель даних. Нижче наведено детальний опис основних сутностей:

| Сутність (Таблиця) | Поля (Атрибути) | Тип даних | Опис |
| :--- | :--- | :--- | :--- |
| **Users** | `user_id` (PK) | UUID | Унікальний ідентифікатор клієнта |
| | `full_name` | VARCHAR(255) | Повне ім'я користувача |
| | `phone` | VARCHAR(15) | Верифікований номер телефону |
| **Accounts** | `account_id` (PK) | VARCHAR(29) | Номер рахунку в форматі IBAN |
| | `user_id` (FK) | UUID | Зв'язок з власником рахунку |
| | `balance` | DECIMAL(15,2) | Поточний залишок (до 2-х знаків) |
| **Transactions** | `transaction_id` (PK)| SERIAL | Автоінкрементний номер операції |
| | `amount` | DECIMAL(15,2) | Сума транзакції |
| | `timestamp` | DATETIME | Дата та час проведення |



### 7.2. Формат обміну даними (JSON Example)
Взаємодія між фронтендом та бекендом для отримання інформації про транзакцію відбувається у форматі JSON:

```json
{
  "transaction_id": 98452,
  "from_account": "UA1234567890...",
  "to_account": "UA0987654321...",
  "amount": 1500.00,
  "currency": "UAH",
  "status": "completed"
}
